// 02 — Use Cases

Real-World Applications

Designed to solve enterprise-scale cloud security challenges across industries handling sensitive data and regulated workloads.

UC-01

Multi-Account Governance

Enterprises managing dozens of AWS accounts can enforce uniform security baselines via SCPs, ensuring no team can bypass security controls regardless of their IAM permissions.

UC-02

Automated Compliance Enforcement

Organizations subject to PCI DSS, NIST 800-53, or SOC 2 audits can demonstrate continuous compliance through automated Config rules and remediation evidence trails.

UC-03

Insider Threat Detection

GuardDuty continuously monitors for anomalous API calls, privilege escalation, and unusual data access patterns — triggering automated alerts and responses.

UC-04

Rapid Secure Account Vending

New product teams can receive a fully secured, pre-configured AWS account with guardrails already in place — reducing onboarding time from weeks to hours.

UC-05

Security Incident Auto-Response

When a violation is detected — such as an exposed S3 bucket or root account login — EventBridge triggers Lambda functions to isolate resources and notify security teams automatically.

UC-06

Regulated Industry Workloads

Financial services, healthcare, and government workloads require strict data residency and encryption controls — all enforced at the organizational policy level through this landing zone.

// 03 — Guardrails

Security Control Catalogue

5 SCPs and Config rules enforced across all OUs. Preventive controls block actions; detective controls alert and remediate.

DENY-ROOT-ACCESS

Blocks all root account API activity across member accounts.

DENY-REGION-RESTRICTION

Restricts resource creation to approved AWS regions only.

DENY-S3-PUBLIC-ACL

Prevents setting public ACLs on any S3 bucket across the org.

DENY-UNENCRYPTED-EBS

Blocks launch of EC2 instances with unencrypted EBS volumes.

DENY-DISABLE-GUARDDUTY

Prevents any principal from disabling GuardDuty in any account.

// 04 — Compliance

Regulatory Frameworks

Controls mapped to four major compliance frameworks, with automated evidence collection and findings surfaced in Security Hub.

CIS AWS

CIS Benchmarks

Center for Internet Security AWS Foundations benchmark controls for identity, logging, monitoring, and networking.

PCI DSS v3.2.1

Payment Card Industry controls for cardholder data protection, access control, and audit logging.

NIST 800-53

NIST 800-53 Rev 5

Federal security and privacy controls covering access control, audit, incident response, and system protection.

SOC 2

SOC 2 Type II

Trust service criteria covering security, availability, and confidentiality with continuous control monitoring.
